# C6288 16x16 multiplier — baseline (added from the VACASK tree)

VACASK's flagship benchmark ([benchmark/c6288](https://codeberg.org/arpadbuermen/VACASK/src/branch/main/benchmark/c6288)):
a medium digital circuit, **10112 transistors / 25380 nodes**, PSP103.4 MOSFETs,
computing 0xFFFF × 0xFFFF = 0xFFFE0001 as a purely analog transistor-level
transient. Each engine's shipped `runme` is tuned to ~1020 timepoints so the
per-timestep cost on a large matrix is what's compared.

**Baseline = each engine's native transistor-level PSP103 run** (no behavioral
substitution; that is the `-bfit` lane, added later). Full-process wall-clock,
1 warm + 2 timed runs, min. Single-threaded, KLU solver, no result files.
Box: P620A (this machine), Linux/WSL2.

| Engine  | Wall (s) | Timepoints (acc/rej) | NR iters | Model path |
| :------ | -------: | :------------------- | -------: | :--------- |
| ngspice-45.2 | 45.98 | 1020 / 1 | 3474 | psp103.va → OSDI 0.3 (classic OpenVAF) |
| VACASK 0.3.3 | 70.08 | 1023 / 10 | 3512 | psp103.va → OSDI 0.4 (OpenVAF-reloaded) |
| Xyce 7.10.0 (our build) | — | — | — | **no native PSP103 / no OSDI** — see below |

**Xyce.** Our Xyce build (DEVELOPMENT 7.10.0-219-g570073b8, the smp-load
branch) has no built-in PSP103 (`level=103` unknown — only MOSFET levels
1/2/3/6, BSIM3/4) and no OSDI loader, so it cannot run this netlist natively.
Bringing C6288 to Xyce needs the PSP103 Verilog-A through **PyMS** (`.hdl`), or
the **`-bfit`** behavioral-substitution lane — tracked separately, not part of
this baseline.

**Reference (VACASK README, AMD Threadripper 7970, stock engines):**
Xyce 151.57 s · ngspice 71.81 s · VACASK 57.98 s.

**Note on ordering.** On the README's Zen4 box VACASK leads. On this box VACASK
and ngspice are closer / reversed — the reference machine has AVX-512, which the
OSDI-compiled model path exploits; this box does not, which narrows VACASK's
edge. Same circuit, matched timepoints, so the comparison is fair.
