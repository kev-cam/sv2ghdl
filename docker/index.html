sv2ghdl build container

This container holds the sv2ghdl source plus the apt-installable build dependencies for the simulation stack. The heavy components (iverilog, NVC, GHDL, Yosys) are built on demand inside the container and exported to the host.

Image is OCI-compliant — works with either docker or podman. Examples below show docker; substitute podman verbatim if you prefer rootless.

0. Build and run the container

docker build -t sv2ghdl-base -f docker/Dockerfile .
docker run -d --name sv2ghdl -p 2222:22 -p 8080:80 sv2ghdl-base

1. Build the stack inside the container

ssh -p 2222 root@localhost /opt/sv2ghdl/docker/build_stack.sh

Default password: sv2ghdl. Output lands in /home/claude/sv2ghdl-stack/usr, structured to mirror a host /usr/local.

To also build Trilinos 14.4 + Xyce (adds ~30–90 min and several GB), set BUILD_XYCE=1:

ssh -p 2222 root@localhost BUILD_XYCE=1 /opt/sv2ghdl/docker/build_stack.sh

2. Export the build to the host

rsync -av -e 'ssh -p 2222' \
    root@localhost:/home/claude/sv2ghdl-stack/usr/ /tmp/sv2ghdl-export/
sudo rsync -a /tmp/sv2ghdl-export/ /usr/local/

3. Verify on the host

which iverilog nvc ghdl yosys
nvc --version
iverilog-sv2ghdl --help

What's inside

  • /opt/sv2ghdl — sv2ghdl source (mounted in at image build)
  • /home/claude/sv2ghdl-stack/src — upstream sources cloned by build_stack.sh
  • /home/claude/sv2ghdl-stack/usr — installed binaries / libs / headers
Security: the default root password is the literal string sv2ghdl. Change it before exposing the SSH port to anything other than localhost.

Repos pulled by build_stack.sh

  • github.com/kev-cam/iverilog — iverilog with VHDL backend
  • github.com/kev-cam/nvc — NVC with --std=2040 + sv2vhdl resolver
  • github.com/kev-cam/ghdl — GHDL fork
  • github.com/YosysHQ/yosys — Yosys (upstream)
